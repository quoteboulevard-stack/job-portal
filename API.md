# Job Portal — Cloud Functions API

Base URL: `https://us-central1-{PROJECT_ID}.cloudfunctions.net`

---

## Authentication

| Type | Mechanism |
|------|-----------|
| Callable functions | Firebase ID token — `Authorization: Bearer <idToken>` (handled by Firebase SDK) |
| Stripe webhook | `Stripe-Signature` header — HMAC-SHA256 of raw body |
| Razorpay webhook | `X-Razorpay-Signature` header — HMAC-SHA256 of raw body |

---

## Rate Limits

No custom rate limiting is configured. Firebase Cloud Functions enforces a default of **1,000 concurrent invocations** per project. Callable functions additionally inherit Firebase's abuse-protection quotas.

---

## Callable Functions

Called via the Firebase SDK (`functions().httpsCallable(name)`). All require a valid Firebase ID token.

### `acceptMessage`

Accept a pending message request. Deducts 1 credit from the job seeker and opens a 30-day conversation.

**Request**
```json
{ "messageId": "string" }
```

**Response**
```json
{ "success": true, "messageId": "string", "timestamp": "<server timestamp>" }
```

**Errors**

| Code | Condition |
|------|-----------|
| `unauthenticated` | No valid ID token |
| `invalid-argument` | `messageId` missing or blank |
| `not-found` | Message or job seeker profile does not exist |
| `permission-denied` | Caller is not the message recipient |
| `already-exists` | Message already accepted |
| `failed-precondition` | Message expired, or job seeker has insufficient credits |

---

### `rejectMessage`

Reject a pending message request. Deducts 1 credit from the job seeker and blocks the sender for 30 days.

**Request**
```json
{ "messageId": "string", "reason": "string (max 500 chars)" }
```

**Response**
```json
{ "success": true, "messageId": "string", "timestamp": "<server timestamp>" }
```

**Errors**

| Code | Condition |
|------|-----------|
| `unauthenticated` | No valid ID token |
| `invalid-argument` | `messageId` or `reason` missing/blank; `reason` exceeds 500 chars |
| `not-found` | Message or job seeker profile does not exist |
| `permission-denied` | Caller is not the message recipient |
| `already-exists` | Message already rejected |
| `failed-precondition` | Message expired, or job seeker has insufficient credits |

---

### `missingSkills`

Analyze skill gaps for an application using Claude AI. **Requires a `premium` custom claim** on the caller's Firebase token.

**Request**
```json
{ "applicationId": "string", "jobId": "string" }
```

**Response**
```json
{
  "skill_gaps": [
    {
      "skill": "Kubernetes",
      "learn_time_months": 3,
      "resources": ["kubernetes.io/docs", "CKAD course"],
      "job_impact": "Required for container orchestration tasks"
    }
  ]
}
```

**Errors**

| Code | Condition |
|------|-----------|
| `unauthenticated` | No valid ID token |
| `invalid-argument` | `applicationId` or `jobId` missing |
| `permission-denied` | Caller lacks `premium` claim, or does not own the application |
| `not-found` | Application does not exist |
| `failed-precondition` | Fit score not yet computed for this application |
| `internal` | Claude analysis failed |

## Webhook Endpoints

Called by payment providers. Not authenticated with Firebase tokens — verified by signature header.

### `POST /stripeWebhook`

Fulfills credits on successful payment. Idempotent — duplicate `payment_intent.succeeded` events are safely ignored.

**Headers**
```
Content-Type: application/json
Stripe-Signature: <generated by Stripe>
```

**Handled event:** `payment_intent.succeeded`

Required `metadata` on the PaymentIntent:

| Field | Type | Description |
|-------|------|-------------|
| `userId` | string | Firebase UID of the buyer |
| `credits` | string | Integer number of credits to grant |
| `plan` | string | Plan name (e.g. `"starter"`) |

**Responses**

| Status | Body | Meaning |
|--------|------|---------|
| 200 | `OK` | Credits fulfilled (or duplicate, safely skipped) |
| 200 | `Event ignored` | Event type not handled |
| 400 | error message | Missing signature or metadata incomplete |
| 500 | `Fulfillment error` | DB write failed — Stripe will retry |

---

### `POST /razorpayWebhook`

Fulfills credits after a Razorpay payment is captured. Idempotent — duplicate `payment.captured` events are safely ignored.

**Headers**
```
Content-Type: application/json
X-Razorpay-Signature: <generated by Razorpay>
```

**Handled event:** `payment.captured`

Required `notes` on the Razorpay order:

| Field | Type | Description |
|-------|------|-------------|
| `userId` | string | Firebase UID of the buyer |
| `credits` | string | Integer number of credits to grant |
| `plan` | string | Plan name (optional) |

**Responses**

| Status | Body | Meaning |
|--------|------|---------|
| 200 | `OK` | Credits fulfilled (or duplicate, safely skipped) |
| 200 | `Event ignored` | Event type not handled |
| 400 | JSON error | Missing signature, or `userId`/`credits` absent or invalid |
| 500 | `Fulfillment error` | DB write failed — Razorpay will retry |

## Internal Triggers (not client-callable)

These functions are invoked automatically by Firestore events or a cron schedule.

| Function | Trigger | Description |
|----------|---------|-------------|
| `sendMessage` | Firestore onCreate `messages/{id}` | Validates participants, sets 7-day expiry, sends email |
| `fitScore` | Firestore onCreate `applications/{id}` | Computes AI fit score via Claude |
| `parseResume` | Storage onFinalize `resumes/{uid}/**` | Extracts and parses resume with Claude |
| `parseJD` | Storage onFinalize `jobs/{jobId}/**` | Extracts and parses job description with Claude |
| `deductCredit` | Firestore onUpdate `messages/{id}` (sent→seen) | Deducts 1 credit from sender |
| `refundCredit` | Firestore onUpdate `messages/{id}` (sent→expired) | Refunds 1 credit to sender |
| `refundExpiredMessages` | Cron — daily 02:00 UTC | Bulk-refunds all messages unseen after 7 days |
| `cleanupOldData` | Cron — Sunday 02:00 UTC | Archives and deletes stale conversations, messages, uploads |
